-- ในสัปดาห์นี้เนื้อหาหลักจะเป็นการทำระบบ Quote Manager ที่มีโครงสร้างรองรับ CRUD เต็มรูปแบบ 
ซึ่งอาจารย์เน้นการแยกส่วนต่าง ๆ ของการทำงานให้ชัดเจนไม่ว่าจะเป็นส่วนที่ใช้ติดต่อกับ API, ส่วนที่ใช้สร้าง DOM, และส่วนที่ใช้จัดการ Error 
เพื่อให้เห็นภาพการทำงานของเว็บแอปครบวงจร โดยใช้ Backend จำลองเพื่อให้ทดลองจริงได้ง่ายขึ้น 

-- ส่วนนี้จะเป็นไฟล์ตัวอย่างจาก Intregrated Project  มาประยุกต์ใหม่ โดยเฉพาะการดึงข้อมูลและสร้างตารางแสดงผล ซึ่งสรุปเป็นประเด็นหลักดังนี้
1. การดึงข้อมูล Fetching Data ใน table.js
ฟังก์ชันจะทำการเรียก (fetch) ข้อมูลไปยัง URL ของ API ที่กำหนด (${API_URL}/study-plans) เพื่อขอรายชื่อแผนการเรียนทั้งหมด
ซึ่งหากการเรียก API ไม่สำเร็จ (เช่น Server ส่ง Status Code ที่ไม่ใช่ 200 OK) ระบบจะหยุดทำงานทันทีและโยน Error พร้อมสถานะโค้ดและข้อความ (throw new Error(...))
2. การสร้างและแสดงตาราง DOM Manipulation
ถ้าการเรียก API สำเร็จ ข้อมูล JSON ที่ได้จะถูกเก็บในตัวแปร rows
โค้ดจะวนลูปผ่านข้อมูลแต่ละแถว (rows.forEach((row) => {...})) 
เพื่อสร้างองค์ประกอบ HTML สำหรับแต่ละแผนการเรียน
3. การจัดการกรณีพิเศษ
หาก API ตอบกลับมาแต่ไม่มีข้อมูลในรายการ (!rows.length) ระบบจะสร้างข้อความ "No data" แสดงในตารางแทน
และหากเกิด Error ขึ้นในกระบวนการ (เช่น เชื่อมต่อไม่ได้ หรือ Server Error) ระบบจะทำการ
บันทึก Error ลงใน Console.
สร้าง Dialog ขึ้นมา และแสดงข้อความเตือนผู้ใช้ว่า "There is a problem. Please try again later."
